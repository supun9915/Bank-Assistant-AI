## 🚀 What This Project Can Do

The **Smart Banking Assistant** is an AI-powered conversational chatbot backend that allows bank customers to interact with their banking data via natural language. Here is a full list of capabilities:

### 💬 Conversational Capabilities

| Feature                           | Description                                                                                             |
| --------------------------------- | ------------------------------------------------------------------------------------------------------- |
| **Greeting & Conversation**       | Recognizes common greetings and responds conversationally with varied short replies                     |
| **Account Balance Inquiry**       | Fetches real-time account balance after identity verification                                           |
| **Transaction History**           | Retrieves and formats the last N (default: 5) transactions with date, amount, type and description      |
| **Loan & Credit Information**     | Provides details on personal loans, home loans, car loans, gold loans, pawning, and fixed deposits      |
| **Account Services**              | Covers account opening, joint accounts, and required documentation                                      |
| **Security & Fraud**              | Guides users through password reset, lost card, fraud reporting, and OTP setup                          |
| **Transfers & Payments**          | Information on local/international transfers, SWIFT, bill payments, and standing orders                 |
| **Fees & Limits**                 | Details on transaction fees, account maintenance fees, and daily limits                                 |
| **Digital Banking**               | Mobile app support, online banking registration, e-statements, and push notifications                   |
| **General Banking Q&A**           | Branch hours, ATM locations, contact numbers, and general enquiries                                     |
| **Cards**                         | Credit/debit card information, rewards, and card management                                             |
| **Investments**                   | Unit trusts, savings plans, and investment product information                                          |
| **Credit Score**                  | Credit score enquiry guidance and tips for improvement                                                  |
| **Complaints**                    | How to raise complaints, escalation paths, and resolution timelines                                     |
| **Foreign Exchange**              | Currency exchange rates, forex transfers, and travel card information                                   |
| **Fixed Deposits**                | FD rates, tenures, maturity options, and premature withdrawal policy                                    |
| **Goodbye**                       | Farewell messages with a friendly sign-off                                                              |
| **Action Request Detection**      | Detects when users want to perform an action (open account, apply for loan) and redirects appropriately |
| **Context-Aware Follow-ups**      | Remembers previous intent to give smart follow-up answers within a conversation turn                    |
| **Unknown Query Handling**        | Saves unrecognised questions with deduplication for future model improvement                            |
| **Account Identity Verification** | OTP-based 3-factor verification (email + National ID + account number) before exposing personal data    |
| **Chat Log Persistence**          | Every request and response is saved to the database for audit and review                                |

### 🧠 NLP Capabilities

- **Primary**: TensorFlow/Keras ANN softmax classifier — trained on intent patterns, produces probability scores per class
- **Hybrid Fallback**: If ANN confidence < 0.40, NLTK keyword matching is used; if both agree on the same intent AND ANN ≥ 0.25, the intent is returned with ANN confidence; otherwise keyword result is used
- **Language Guard**: Unicode range check (Sinhala, Arabic, CJK, Devanagari, Cyrillic, etc.) + `langdetect` for Latin-script non-English text — non-English input returns `UNSUPPORTED_LANGUAGE` before NLP
- Tokenises and Lancaster-stems user input; encodes as bag-of-words vector before ANN inference
- Confidence threshold of **0.40** — predictions below this fall through to hybrid keyword matching
- Nonsense/non-text guard — inputs with fewer than 2 alphabetic characters return UNKNOWN immediately (no model inference)
- `reload_intents()` function allows hot-reloading of intent definitions at runtime
- Regex-based entity extraction for numeric and currency values

### 🌐 API Capabilities

- RESTful HTTP API built on FastAPI
- Interactive auto-generated documentation at `/docs` (Swagger UI) and `/redoc`
- Health check endpoints (`/` and `/health`) for monitoring
- CORS-enabled for browser-based frontends
- Structured JSON responses with `reply`, `intent`, `confidence`, and optional `data` fields
- OTP-based account verification endpoints for secure personal data access

### 🔐 Account Verification Flow

1. Frontend calls `POST /api/account/send-otp` with email, National ID, and account number
2. Backend validates credentials against the database
3. A 6-digit OTP is generated, stored in memory (5-minute expiry), and emailed to the customer
4. Frontend calls `POST /api/account/verify-otp` with the OTP
5. On success, `user_id` and `account_number` are returned to the frontend
6. Subsequent `/api/chat` requests carry `account_number` to unlock personal data intents (BALANCE, TRANSACTIONS)

## 📋 File Descriptions

### Core Application Files

#### `main.py`

- FastAPI application initialization
- CORS middleware configuration
- Route registration for `/api/chat` and `/api/account`
- Logging setup
- Health check endpoints

**Key Components:**

- `app` — FastAPI instance with title, description, version
- `@app.get("/")` — Root health check
- `@app.get("/health")` — Detailed health check

---

#### `db.py`

- Database connection management via context manager
- All SQL query functions
- Parameterised queries throughout (SQL-injection safe)

**Key Functions:**

- `get_db_connection()` — Connection context manager
- `execute_query()` — Generic query executor
- `get_user_by_id()` — Fetch user row by ID
- `get_user_by_email_and_account()` — Validate identity (email + National ID + account number)
- `save_verified_user()` — Record successful OTP verification
- `get_account_balance()` — Fetch user balance
- `get_recent_transactions()` — Fetch transaction history
- `save_chat_log()` — Persist every request/response to `chat_logs`
- `save_unknown_question()` — Store unknown queries with deduplication
- `get_all_unknown_questions()` — Retrieve all saved unknown queries
- `test_connection()` — Database health check

**Database Tables Used:**

- `users` — User accounts (includes `id_number` column)
- `accounts` — Bank accounts
- `transactions` — Transaction history
- `chat_logs` — Audit log of all chatbot interactions
- `unknown_questions` — Self-learning dataset
- `verified_users` — OTP verification records
- `loans` — Customer loan records
- `loan_repayments` — Loan repayment schedules
- `fixed_deposits` — Fixed deposit records
- `pawning` — Pawning/gold loan records

---

#### `nlp.py`

- Natural Language Processing — ANN primary, keyword fallback
- Intent detection with confidence scoring

**Key Functions:**

- `preprocess_text()` — Lowercase, strip, normalise whitespace
- `_tokenize_and_stem()` — Lancaster-stem alpha tokens
- `_bag_of_words()` — Encode text as binary BoW vector
- `_has_meaningful_text()` — Guard: reject inputs with < 2 alphabetic chars
- `_keyword_fallback()` — Weighted keyword-matching fallback
- `detect_intent()` — Primary ANN inference; falls back to keywords; returns `(intent, confidence)`
- `extract_entities()` — Regex extraction of numbers and currency values
- `reload_intents()` — Hot-reload intent definitions from `intents.json`

**Supported Intents (22 total):**

- `GREETING`, `GOODBYE`, `BALANCE`, `TRANSACTIONS`, `LOAN`, `ACCOUNT_SERVICES`
- `SECURITY`, `TRANSFERS`, `FEES`, `DIGITAL_BANKING`, `GENERAL`
- `CARDS`, `INVESTMENTS`, `CREDIT_SCORE`, `COMPLAINTS`
- `FOREIGN_EXCHANGE`, `FIXED_DEPOSIT`, `PAWNING`
- `FORGOT_EMAIL`, `PROFANITY_RESPONSE`, `CAPABILITIES`, `UNKNOWN`

**ANN Configuration:**

- Model file: `models/chatbot_model.keras`
- Vocabulary: `models/words.pkl` (Lancaster-stemmed)
- Class labels: `models/classes.pkl`
- Confidence threshold: **0.40** (hybrid keyword fallback if below this)
- Training: 1142 samples, stratified split (85/15), balanced class weights

---

#### `train_model.py`

- Trains the Keras ANN intent classifier
- Reads patterns from `intents/intents.json`
- Tokenises, stems, and encodes patterns into bag-of-words vectors
- Saves `chatbot_model.keras`, `words.pkl`, `classes.pkl` to `models/`
- Run once before starting the server: `python train_model.py`

---

#### `migrate.py`

- CLI tool for applying versioned SQL migrations
- Reads files from `migrations/` matching `V###__name.sql`
- Tracks applied migrations in a `_migrations` table in MySQL
- Auto-creates the database if it does not exist

**Usage:**

```bash
python migrate.py up              # Apply all pending migrations
python migrate.py status          # Show applied / pending migrations
python migrate.py create <name>   # Scaffold a new migration file
```

### Models Directory (`models/`)

#### `chat_models.py`

Pydantic models for request/response validation.

**Classes:**

- `ChatRequest` — Validates incoming chat messages
  - `message: str` — User input (required, 1–500 chars)
  - `user_id: int` — User identifier (optional, default=1)
  - `last_intent: str` — Intent from previous turn for context-aware follow-ups (optional)
  - `account_number: str` — Verified account number to unlock personal data intents (optional)

- `ChatResponse` — Structures bot responses
  - `reply: str` — Bot's response message
  - `intent: str` — Detected intent
  - `confidence: float` — Confidence score (0.0 – 1.0)
  - `data: dict` — Additional data (e.g., balance, transactions) (optional)

## 🎯 Design Patterns Used

### 1. **Layered Architecture**

```
Presentation Layer (routes/)
    ↓
Business Logic Layer (services/)
    ↓
Data Access Layer (db.py)
    ↓
Database (MySQL)
```

### 2. **Strategy Pattern**

- Different intent handlers swapped in based on detected intent
- ANN / keyword fallback strategy in `nlp.py`

### 3. **Context Manager Pattern**

- Database connections use `@contextmanager` for automatic cleanup

### 4. **Guard Clause Pattern**

- `_has_meaningful_text()` rejects non-text input early
- `_require_account_selection()` blocks personal data access before OTP verification

### 5. **Service Layer Pattern**

- Business logic separated from API routes
- Reusable service functions callable independently of HTTP layer

---

## 🔐 Security Features

1. **Input Validation**
   - Pydantic models enforce types and length constraints on all inputs
   - `_has_meaningful_text()` rejects numeric/symbol-only messages before ANN inference

2. **SQL Injection Prevention**
   - All queries use parameterised placeholders — no string concatenation

3. **OTP-Based Identity Verification**
   - 3-factor check: email + National ID + account number before OTP is issued
   - OTP is 6-digit numeric, valid for 5 minutes, stored server-side only
   - Personal intents (BALANCE, TRANSACTIONS) blocked until `account_number` is verified

4. **Language Guard**
   - Step 1: Unicode range detection rejects Sinhala, Arabic, CJK, Devanagari, Cyrillic, and 20+ other non-Latin scripts before any NLP processing
   - Step 2: `langdetect` identifies Latin-script non-English (French, Spanish, etc.) for messages with 3+ words
   - Returns `UNSUPPORTED_LANGUAGE` with an English-only message

5. **Secret Management**
   - Database credentials and email app-passwords loaded from `.env` only
   - `.env` is excluded from version control via `.gitignore`

6. **Error Handling**
   - Try/except blocks at every layer; errors logged, not exposed in responses
   - No sensitive data in HTTP error messages

7. **Connection Management**
   - Database connections closed automatically via context manager even on exceptions

## 🚀 Development Workflow

1. **Setup**

   ```bash
   ./setup.bat  # or ./setup.sh
   ```

2. **Configure**

   ```bash
   # Edit .env with DB credentials and email settings
   ```

3. **Run Migrations**

   ```bash
   python migrate.py up
   ```

4. **Train ANN Model** _(once, or after adding new intents)_

   ```bash
   python train_model.py
   ```

5. **Run Server**

   ```bash
   uvicorn main:app --reload
   ```

6. **Test**

   ```bash
   python test_chatbot.py
   # or use Postman collection
   # or browse /docs for Swagger UI
   ```

7. **Develop**
   - Add new intents in `intents/intents.json` (patterns + keywords)
   - Re-run `python train_model.py` to rebuild the ANN
   - Add handlers in `services/chat_service.py`
   - Add routes in `routes/`
   - Add database functions in `db.py`
   - Add SQL migrations in `migrations/` and run `python migrate.py up`
